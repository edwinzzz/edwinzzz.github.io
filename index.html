19. Remove Nth Node From End of List

Given a linked list, remove the nth node from the end of list and return its head.

关键点

  • 删除节点使用哪种方法？

    本题中可以拿到节点的value和next两个信息，所以理论上可以使用 O(1)方法，但是可能需要删除尾节点，所以O(1)方法不适用。

  • 循环条件的开闭如何确定？

    从条件来看，n为倒数第n个节点，而不是倒数index = n的节点，我们可以确定目标节点需要通过n-1次next操作才能变成尾节点。这意味着当fast指针先进行n-1次next操作，当fast达到尾节点的时候，slow指针达到目标节点；如果fast指针先进行n此next操作，那么当它达到尾节点的时候，slow指针到达目标节点的前一个节点。

  • 如何判断fast指针到达尾节点

    尾节点的特性是next指针为空，所以while循环条件应该是next指针不为空

系统对于weak变量维护着一个weak表，该表把对象的地址的hash值作为key，把指向该对象的weak变量的地址作为value。objc_storeWeak指令执行的操作即是以其第二个参数的hash值作为key，将第一个参数注册到其对应的value中。如果第二个参数的值为0，则把weak变量置为nil，并从weak表中删除。

声明类型

捕捉自动变量

block捕捉到了main函数上下文中的自动变量a

  • 只捕捉自动变量的瞬间值
  • 不能修改捕捉的自动变量
  • 不能赋值给捕捉到的对象
  • 可以使用捕捉到的对象

处理全局变量、静态全局变量和静态局部变量

  • 全局变量和静态全局变量都和函数一样直接访问
  • 静态局部变量是捕捉其指针，对其进行访问

__block修饰符

  • __block修饰符将自动变量转化为结构体

上述变量val转化为了下面的结构体

  • __block修饰符变量产生的结构体会在捕获其的Block被复制到堆上时一并被复制到堆上，并被其持有，并且在栈上的结构体和被复制到堆上的结构的forwarding指针均指向堆上结构体的内存地址

Block类型

  • _NSConcretStackBlock(栈Block)
  • _NSConcretMallocBlock(堆Block)
  • _NSConcretGlobalBlock(全局Block)

区别

  • 不截获自动变量的Block为全局Block
  • 截获自动变量的Block为栈Block
  • 栈Block复制后成为堆Block

Block循环应用的解决方法

  • 声明附有__weak修饰符的变量
  • 使用附有__block修饰符的变量
  • 使用__block变量的优点
    • 通过__block变量可控制对象的持有时间
    • 在不能使用_weak修饰符的环境中可以不使用__unsafe_unretain
  • 使用__block变量的缺点
    • 为了避免循环引用必须执行block
